## ADDED Requirements

### Requirement: 全屏视图内新建文章
用户 MUST 能够在全屏文档编辑视图内直接新建文章，无需跳转到其他页面。新建文章在首次保存前仅存在于前端状态，不会创建数据库记录。

#### Scenario: 点击新建按钮进入新建模式
- **GIVEN** 用户已登录并在 `/posts-view` 页面
- **WHEN** 用户点击左侧侧边栏的「新建」按钮
- **THEN** 系统进入「新建模式」（纯前端状态）
- **AND** 右侧编辑器显示空白文章（标题为空，内容为空）
- **AND** 左侧文档列表保持不变（新文章尚未创建）
- **AND** 不调用任何后端 API

#### Scenario: 新建模式下首次保存文章
- **GIVEN** 用户处于新建模式并已编辑文章内容
- **WHEN** 用户点击保存按钮
- **THEN** 系统调用 `createPost` API 创建文章
- **AND** 创建成功后切换为「编辑模式」
- **AND** 新文章出现在左侧文档列表中
- **AND** 新文章被自动选中

#### Scenario: 新建文章时有未保存的更改
- **GIVEN** 用户正在编辑一篇文章且有未保存的更改
- **WHEN** 用户点击「新建」按钮
- **THEN** 系统显示确认对话框提示用户当前文档有未保存的更改
- **AND** 若用户确认，则进入新建模式
- **AND** 若用户取消，则保持当前文档不变

#### Scenario: 新建模式下切换到其他文章
- **GIVEN** 用户处于新建模式并已编辑文章内容（`isDirty = true`）
- **WHEN** 用户点击左侧文档列表中的某篇文章
- **THEN** 系统显示确认对话框提示用户当前文档有未保存的更改
- **AND** 若用户确认，则放弃新建内容并切换到选中文章
- **AND** 若用户取消，则保持新建模式不变

#### Scenario: 新建文章保存失败
- **GIVEN** 用户处于新建模式并点击保存
- **WHEN** `createPost` API 调用失败
- **THEN** 系统显示错误提示 Toast
- **AND** 保持新建模式和当前编辑内容不变
