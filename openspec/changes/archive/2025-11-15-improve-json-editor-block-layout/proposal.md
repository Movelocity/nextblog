# Change: JSON 编辑器优化布局（限制最多3个block）

## Why
当前 JSON 编辑器允许创建无限多个编辑框，导致以下问题：

1. **页面混乱**：过多的编辑框堆叠在一起，需要大量滚动
2. **性能下降**：太多编辑器实例会影响性能
3. **使用场景有限**：实际上大多数场景只需要 1-3 个编辑框

用户实际使用场景：
- **单任务场景**：只需一个编辑框进行 JSON 格式化、文本处理等操作
- **对比场景**：需要 2-3 个编辑框并排对比数据、转换前后等

简化方案：
- 限制最多创建 3 个编辑框
- 响应式布局：屏幕宽度不够时自动折叠为更少的列
- 保持简单，不引入复杂的标签页系统

## What Changes
- **限制编辑框数量**：最多只能创建 3 个编辑框
- **响应式布局优化**：
  - 桌面（>1024px）：自动根据编辑框数量显示 1-3 列
  - 平板（640-1024px）：最多显示 2 列
  - 移动端（<640px）：强制单列显示
- **保持现有功能**：
  - 保留编辑框的尺寸调整功能（垂直拖动）
  - 保留所有文本操作和自定义脚本功能
  - 保留 IndexedDB 存储

## Impact
- **受影响的规格**: `json-editor`
- **受影响的代码**: 
  - `app/(views)/tools/json/page.tsx` - 主编辑器页面（添加数量限制和布局逻辑）
- **用户体验**: 
  - **简化界面**：不再有无限多的编辑框
  - **性能提升**：最多 3 个编辑器实例
  - **更好的响应式**：自动适应屏幕大小
- **向后兼容**: 部分兼容
  - 如果用户已有超过 3 个编辑框，只显示前 3 个
  - 所有数据保留，用户可以手动删除多余的编辑框
